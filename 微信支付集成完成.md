## 🔧 技术实现

### 支付流程
```
前端                        后端                    微信支付
  |                          |                         |
  |-- 创建订单 ------------->|                         |
  |                          |-- 调用微信API --------->|
  |                          |<-- 返回code_url --------|
  |<-- 返回二维码 ------------|                         |
  |                          |                         |
  |-- 轮询查询订单状态 ------>|                         |
  |                          |-- 查询订单 ------------>|
  |                          |<-- 返回支付状态 --------|
  |<-- 返回是否已支付 --------|                         |
  |                          |                         |
[用户扫码支付]                |                         |
  |                          |                         |
  |                          |<-- 支付回调 ------------|
  |                          |    (webhook)            |
  |                          |-- 更新数据库 ---------->|
  |                          |                         |
  |-- 轮询查到已支付 -------->|                         |
  |<-- 解锁解卦内容 ----------|                         |
```

### 数据库表
**payment_logs** 表记录所有支付：
- `orderId`: 订单号
- `userId`: 用户ID
- `amount`: 金额（分）
- `status`: pending / success / failed
- `createdAt`: 创建时间

## ⚠️ 注意事项

### 安全
1. ❗ `.env` 文件包含敏感信息，切勿提交到Git
2. ❗ `certs/` 目录的证书文件要妥善保管
3. ❗ 生产环境必须使用HTTPS
4. ❗ 定期备份 `data/app.db` 数据库

### 证书文件
生产环境需要在项目根目录创建 `certs/` 文件夹并放入：
- `apiclient_key.pem` （私钥）
- `apiclient_cert.pem` （证书）

### 环境要求
- Node.js v16+ （当前使用 v24.12.0）
- SQLite 3
- 生产环境需要HTTPS和公网IP
