# 微信支付功能说明

## 功能概述

本应用已集成微信支付功能，用户需要**支付3元人民币**才能获得解卦结果。

## 支付流程

1. 用户完成六爻摇卦后，会显示排盘结果
2. 点击"解卦"按钮，弹出支付弹窗
3. 支付弹窗显示金额：¥3.00
4. 用户点击"生成二维码"按钮
5. 系统生成微信支付二维码
6. 用户使用微信扫码支付
7. 支付成功后自动显示解卦结果

## 两种运行模式

### 1. 模拟支付模式（开发测试）

**适用场景**：本地开发、测试环境

**特点**：
- 不需要配置微信支付
- 点击"生成二维码"后直接模拟支付成功
- 无需实际支付即可看到解卦结果
- 适合开发调试

**使用方法**：
- 不配置微信支付相关环境变量即可
- 或者配置不完整时自动启用

### 2. 真实支付模式（生产环境）

**适用场景**：正式上线运营

**特点**：
- 生成真实的微信支付二维码
- 用户必须实际支付3元
- 支付数据记录到数据库
- 可在管理后台查看支付记录

**使用方法**：
1. 申请微信支付商户号（参见`微信支付配置说明.md`）
2. 下载API证书文件
3. 配置环境变量（参见下方配置说明）

## 配置微信支付

### 步骤1：申请微信支付商户号

1. 访问 https://pay.weixin.qq.com/
2. 注册并完成企业认证
3. 获取以下信息：
   - APPID（应用ID）
   - MCHID（商户号）
   - API v3密钥
   - 证书序列号

### 步骤2：下载API证书

1. 登录微信支付商户平台
2. 进入"账户中心" → "API安全" → "申请API证书"
3. 下载证书文件：
   - `apiclient_key.pem`（私钥）
   - `apiclient_cert.pem`（证书）
4. 在项目根目录创建 `certs` 文件夹
5. 将两个证书文件放入 `certs` 文件夹

### 步骤3：配置环境变量

创建 `.env` 文件（参考 `.env.example`）：

```env
# 微信支付配置
WECHAT_APPID=你的应用ID
WECHAT_MCHID=你的商户号
WECHAT_SERIAL_NO=你的证书序列号
WECHAT_APIV3_KEY=你的APIv3密钥
WECHAT_NOTIFY_URL=https://你的域名.com/api/payment-notify

# 微信支付证书路径
WECHAT_PRIVATE_KEY_PATH=./certs/apiclient_key.pem
WECHAT_PUBLIC_KEY_PATH=./certs/apiclient_cert.pem
```

### 步骤4：配置回调URL

1. 登录微信支付商户平台
2. 进入"产品中心" → "开发配置"
3. 设置支付回调URL为：`https://你的域名.com/api/payment-notify`
4. 确保服务器可以被外网访问（需要HTTPS）

## 支付金额修改

如需修改支付金额，请修改以下文件：

### 前端：`frontend/src/pages/ShakePage.vue`

```vue
<PaymentModal
  v-if="showPayment"
  :amount="3.00"  <!-- 修改这里的金额 -->
  @close="closePaymentModal"
  @success="handlePaymentSuccess"
/>
```

### 后端：`src/server-api.ts`

```typescript
const amount = data.amount || 300; // 默认3元（300分），修改这里
```

**注意**：微信支付金额单位是"分"，所以3元 = 300分

## 数据库记录

支付记录存储在 SQLite 数据库中：

**表名**：`payment_logs`

**字段**：
- `id`：记录ID
- `userId`：用户ID
- `orderId`：订单号
- `amount`：支付金额（分）
- `status`：支付状态（pending/success/failed）
- `createdAt`：创建时间

可在管理后台查看支付记录和统计数据。

## 管理后台

访问：`http://你的域名/admin`

管理后台可以查看：
- 支付统计数据
- 支付记录列表
- 使用记录
- 导出Excel报表

## API接口说明

### 1. 创建支付订单

```
POST /api/create-payment
Content-Type: application/json

{
  "amount": 300,           // 金额（分）
  "description": "六爻解卦服务",
  "userName": "用户名"
}

响应（模拟模式）：
{
  "code": 0,
  "message": "订单创建成功（模拟模式）",
  "orderId": "ORDER_xxx",
  "amount": 300,
  "mockMode": true
}

响应（真实支付）：
{
  "code": 0,
  "message": "订单创建成功",
  "orderId": "ORDER_xxx",
  "amount": 300,
  "qrCode": "data:image/png;base64,xxx",  // 二维码图片
  "codeUrl": "weixin://xxx"               // 微信支付链接
}
```

### 2. 查询订单状态

```
GET /api/query-order?orderId=ORDER_xxx

响应：
{
  "code": 0,
  "isPaid": true,          // 是否已支付
  "tradeState": "SUCCESS", // 交易状态
  "message": "支付成功"
}
```

### 3. 微信支付回调通知

```
POST /api/payment-notify

微信支付服务器会自动调用此接口，无需手动调用
```

## 安全注意事项

1. ⚠️ **证书文件安全**
   - 证书文件包含敏感信息，切勿上传到代码仓库
   - `.gitignore` 已配置忽略 `certs/` 目录

2. ⚠️ **环境变量安全**
   - `.env` 文件包含密钥信息，切勿提交到Git
   - `.gitignore` 已配置忽略 `.env` 文件

3. ⚠️ **HTTPS要求**
   - 生产环境必须使用HTTPS
   - 微信支付回调URL必须是公网可访问的HTTPS地址

4. ⚠️ **回调验证**
   - 系统会自动验证微信支付回调签名
   - 确保不要禁用签名验证功能

## 故障排查

### 问题1：启动时显示"微信支付未配置"

**原因**：环境变量未配置或配置不完整

**解决**：
- 检查 `.env` 文件是否存在
- 检查所有微信支付相关环境变量是否配置
- 检查证书文件是否存在于 `certs/` 目录

### 问题2：生成二维码失败

**原因**：微信支付API调用失败

**解决**：
- 检查网络连接
- 检查APPID、MCHID等配置是否正确
- 检查证书文件路径是否正确
- 查看控制台错误日志

### 问题3：支付成功但未显示结果

**原因**：支付回调未收到或处理失败

**解决**：
- 检查回调URL配置是否正确
- 检查服务器是否可被外网访问
- 检查HTTPS证书是否有效
- 查看服务器日志

### 问题4：模拟支付模式无法切换到真实支付

**原因**：配置检查失败

**解决**：
- 确保所有环境变量都已配置
- 确保证书文件存在且路径正确
- 重启服务器使配置生效
- 查看启动日志：
  - `✓ 微信支付已启用` - 真实支付模式
  - `⚠ 微信支付未配置` - 模拟支付模式
