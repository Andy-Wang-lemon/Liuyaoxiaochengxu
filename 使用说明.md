# 六爻解卦应用 - 微信支付版使用说明

## 🎉 功能介绍

本应用已成功集成微信支付功能！用户需要**支付3元**才能查看AI解卦结果。

## 🚀 快速启动

### 1. 开发测试模式（模拟支付）

适合本地测试，无需配置微信支付，自动模拟支付成功。

```bash
# 启动服务器
npm run web

# 访问地址
http://localhost:3003
```

在此模式下：
- 点击"生成二维码"后会直接模拟支付成功
- 无需实际支付即可测试完整流程
- 适合开发调试使用

### 2. 生产环境模式（真实支付）

需要配置微信支付，用户必须实际支付3元。

#### 步骤 1：申请微信支付

1. 访问 https://pay.weixin.qq.com/
2. 注册商户号并完成认证
3. 获取以下信息：
   - APPID
   - 商户号 (MCHID)
   - APIv3密钥
   - 证书序列号

#### 步骤 2：下载证书

1. 登录微信支付商户平台
2. 下载 API 证书文件：
   - `apiclient_key.pem`
   - `apiclient_cert.pem`
3. 在项目根目录创建 `certs` 文件夹
4. 将证书文件放入 `certs` 目录

#### 步骤 3：配置环境变量

创建 `.env` 文件：

```env
# Gemini API（必填）
GEMINI_API_KEY=你的Gemini_API密钥

# 微信支付配置
WECHAT_APPID=你的应用ID
WECHAT_MCHID=你的商户号
WECHAT_SERIAL_NO=你的证书序列号
WECHAT_APIV3_KEY=你的APIv3密钥
WECHAT_NOTIFY_URL=https://你的域名.com/api/payment-notify

# 证书路径
WECHAT_PRIVATE_KEY_PATH=./certs/apiclient_key.pem
WECHAT_PUBLIC_KEY_PATH=./certs/apiclient_cert.pem

# 服务器端口
PORT=3003

# 管理后台密码
ADMIN_TOKEN=你的管理员密码
```

#### 步骤 4：配置回调地址

1. 登录微信支付商户平台
2. 设置回调URL：`https://你的域名.com/api/payment-notify`
3. 确保服务器使用 HTTPS 且可被外网访问

#### 步骤 5：启动服务

```bash
# 编译代码
npm run build
npm run build:frontend

# 启动服务
npm run web
```

## 📱 使用流程

1. **填写信息**
   - 起卦地点
   - 时区（默认 Asia/Shanghai）
   - 测算人姓名
   - 所问何事

2. **摇卦**
   - 点击"摇卦"按钮6次
   - 每次都会生成一爻
   - 系统自动生成排盘结果

3. **支付解卦**
   - 点击"解卦"按钮
   - 弹出支付弹窗（¥3.00）
   - **开发模式**：点击"生成二维码"直接成功
   - **生产模式**：扫码支付3元

4. **查看结果**
   - 支付成功后自动显示 AI 解卦结果
   - 可继续追问（每次追问也需支付3元）

## 💰 支付金额修改

如需修改支付金额，请编辑以下文件：

**前端** - `frontend/src/pages/ShakePage.vue`（第165行）：
```vue
<PaymentModal
  v-if="showPayment"
  :amount="3.00"  <!-- 修改这里 -->
  @close="closePaymentModal"
  @success="handlePaymentSuccess"
/>
```

**后端** - `src/server-api.ts`（第280行）：
```typescript
const amount = data.amount || 300; // 3元 = 300分
```

⚠️ 注意：微信支付金额单位是"分"，1元 = 100分

## 🔧 常见问题

### Q1：启动时显示"微信支付未配置"
**A**：这是正常的，表示当前处于模拟支付模式。如需启用真实支付，请按照上述步骤配置微信支付。

### Q2：支付后没有显示结果
**A**：检查浏览器控制台是否有错误，确认后端服务正常运行。

### Q3：如何切换模式？
**A**：
- 移除或不完整配置 `.env` 中的微信支付参数 → 模拟模式
- 完整配置所有微信支付参数 → 真实支付模式

### Q4：生产环境部署注意事项
**A**：
- 必须使用 HTTPS
- 确保回调URL可被外网访问
- 定期备份数据库（`data/app.db`）
- 保护好证书文件和环境变量
